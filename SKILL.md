# IR Chronicle Matrix（三时三事检索矩阵）

## 概述

本 skill 将国际关系研究对象（"主语行为体"）放入一个 3×3 交叉矩阵中做结构化梳理：

- **行（三时 · Chronicles）**：第一时间（起源）、周期时间（发展阶段）、现状时间（当前状态）
- **列（三事 · Multi-Layers）**：上位事件（宏观结构嵌入）、下位事件（具体实践分支）、平行事件（横向比较对象）

矩阵是**地图**，不是百科全书。每格只放最核心的导航信息。用户选定重点交叉点后，以 markdown 正文做深挖报告——有完整空间容纳分级事实、来源、张力分析和信息缺口。

工作分两个 Phase：
- **Phase 1**（必做）：生成矩阵表 + 辅助卡片，用户可选深挖若干单元格
- **Phase 2**（严格可选）：基于矩阵和深挖结果提炼核心线索，生成实证研究选题方向

研究者主导方向，AI 辅助执行。每条事实必须带置信度标签和来源。

## Phase 1：三时三事检索矩阵生成

### Step 1：边界界定

先明确研究对象边界，再开始检索。这是 ir-fact-retrieval 的核心纪律：把问题限定到一个可检索对象。

**默认做的：**
- 明确主语行为体名称
- 框定时间起止、空间范围、核心行为者
- 用 1-2 句话写出本 skill 采用的研究边界

**何时需要问用户：**
- 主语行为体表述宽泛（如"中美关系"）——建议收缩到具体议题、时段或互动维度
- 用户同时混合多个事件/时间段/地区
- 用户把"事实整理"和"责任判断/政策建议"混在一起

若需要收缩，优先建议四个方向：**时间收缩**（只取某个阶段）、**空间收缩**（只取某个地区或双边）、**议题收缩**（只取某个政策领域）、**层次收缩**（只取某个分析层次）。只问最小必要问题，不打断用户做长篇确认。

### Step 2：输出轻量 3×3 矩阵表 + 辅助卡片

一次性输出以下四项内容。

#### 输出项 1：研究边界卡

简短的单行表格，让研究者一眼看到本次检索的口径：

| 时间范围 | 空间/层次 | 议题边界 | 明确排除 |
|---------|----------|---------|---------|
| XXXX 年至今 | 国家/区域/全球层次 | 聚焦的议题范围 | 不覆盖的内容 |

#### 输出项 2：三时三事检索矩阵表

轻量 3×3 表格，每格统一格式。每格内容需具体可操作，不能是泛泛套话。

**表格中每格必须包含**：
- **核心问题**：这个交叉点要回答的那个关键问题（1 句，问句形式）
- **检索关键词**：3-4 个即搜即用的关键词组合（可直接粘贴到搜索引擎或学术数据库）
- **识别目标**：在这个交叉点要找什么信息（1 句）

```
|           | 上位事件 (Extensions) | 下位事件 (Branches) | 平行事件 (Synchronic) |
|-----------|----------------------|---------------------|----------------------|
| **第一时间** | 核心问题：xxx         | 核心问题：xxx        | 核心问题：xxx         |
| (Origin)  | 关键词：xxx; xxx      | 关键词：xxx; xxx     | 关键词：xxx; xxx      |
|           | 识别目标：xxx         | 识别目标：xxx        | 识别目标：xxx         |
| **周期时间** | 核心问题：xxx         | 核心问题：xxx        | 核心问题：xxx         |
| (Stages)  | 关键词：xxx; xxx      | 关键词：xxx; xxx     | 关键词：xxx; xxx      |
|           | 识别目标：xxx         | 识别目标：xxx        | 识别目标：xxx         |
| **现状时间** | 核心问题：xxx         | 核心问题：xxx        | 核心问题：xxx         |
| (Present) | 关键词：xxx; xxx      | 关键词：xxx; xxx     | 关键词：xxx; xxx      |
|           | 识别目标：xxx         | 识别目标：xxx        | 识别目标：xxx         |
```

**表格生成原则：**
- 核心问题必须是问句，且对该交叉点具有特异性——不能是"它是什么"这种泛泛问题
- 关键词必须可直接用于搜索，优先采用"主语行为体 + 维度限定词"的组合式表达
- 识别目标须具体说明要找什么类型的信息（政策文件、官方声明、事件报道、学术分析等）
- 避免空泛、宣传式或口号式表达
- 每个单元格 3-5 行，保持表格可扫读

#### 输出项 3：大事记时间线骨架

5-10 行，覆盖从起源到现状的关键节点：

| 时间 | 事件 | 维度归属 | 置信度 |
|------|------|---------|--------|
| 20XX 年 | 事件描述 | 第一时间/周期时间/现状时间 | [高置信] |
| 20XX 年 | 事件描述 | 周期时间 × 下位事件 | [待核实] |
| ... | ... | ... | [推测节点] |

置信度标签：
- **[高置信]**：有官方文件或多方一致记录支持
- **[待核实]**：来自单一来源或二手资料，需进一步确认
- **[推测节点]**：基于逻辑推断的关键节点位置，尚未找到直接证据

#### 输出项 4：深挖建议

从 9 个单元格中，标注 2-3 个推荐优先深挖的单元格。推荐标准：

1. **张力最强**：该交叉点存在明显反常、政策反转、未解争议或断裂
2. **信息可得性最高**：该交叉点的材料最容易获取，适合优先突破
3. **杠杆效应最大**：理解该交叉点能带动对其他单元格的理解

每个推荐附一句话推荐理由。推荐应建立在对主语行为体具体特质的判断上，不能是"这个维度很重要"式的通用话术。

然后提问：

> 你想让我对哪些单元格进行深度检索？可以直接回复交叉点名称（如"第一时间×下位事件"和"现状时间×上位事件"），也可以输入"全部"或"跳过"。

### Step 3：深挖选中单元格

用户选择后，对每个被选中的单元格，以 **markdown 正文** 形式输出深挖报告。深挖报告在表格下方单独成节，不塞回表格。每个单元格一节，用小标题区分。

深挖前，必须执行事实检索（WebSearch 或知识库检索），不能仅凭模型已有知识生成。

#### 深挖输出格式

```
### 深挖：[单元格名称]，如"第一时间 × 上位事件"

**核心发现**
- [已确认] 事实描述。来源：来源名称 + 日期（如：中国外交部白皮书，2024-03）
- [主流认定] 学界/主要机构的共识判断。来源：来源名称 + 日期
- [存在争议] 争议点描述，说明哪方持什么观点。来源：来源A vs 来源B
- [待核实] 待查事实，说明为什么还不能确认。来源：来源名称 + 日期（如仅有单一来源）

**核心张力**
该交叉点存在什么反常现象、未解问题或矛盾之处？如果当前材料不足以判断，也需明确说明。

**关键行为者与角色**
- 行为者A：角色描述（主导者/回应者/协调者/制约者/外部影响者）
- 行为者B：角色描述

**信息缺口**
- 具体缺什么信息
- 建议去哪里补（具体数据库、文件类型、机构网站等）

**后续检索关键词**
- 关键词组合 1
- 关键词组合 2
```

#### 深挖方法论（继承 ir-fact-retrieval）

**来源优先级（从高到低）：**
1. 官方与原始文件：条约、协定、联合国决议、政府白皮书、外交部声明、法律文本
2. 权威机构与数据库：SIPRI、UCDP、ACLED、World Bank、IMF、IEA 等
3. 主流媒体与学术二手资料：Reuters、AP、BBC、学术期刊、权威智库报告

**事实分级规则：**
- `[已确认]`：有官方文件、条约原文或多方一致记录支持
- `[主流认定]`：学术界、主要国际组织或多家高质量来源形成较稳定共识
- `[存在争议]`：行为者之间、媒体之间或研究者之间存在明显冲突说法；须说明争议双方各自主张
- `[待核实]`：仅见单一来源、证据链不完整，或数据尚未稳定

**纪律：**
- 事实与解释分开标记，不在同一句里混写
- 所有数字必须带时间点，如"截至 2026 年 4 月"
- 若不同来源给出不同数字，并列呈现并解释口径差异，不强行合并
- 争议分"事实争议"（边界、数字、时间、是否违反协议）和"解释争议"（责任归属、法律定性、行为意图）
- 不做立场判断，不替任何一方背书，不滑向政策建议
- 避免把单一媒体报道、匿名消息、社交媒体帖子直接当作已确认事实
- 如果关键信息确实查不到，就标注为 [待核实]，不伪装成完整结论

**深挖篇幅控制：**
- 每个单元格深挖的"核心发现"不超过 4 条
- "信息缺口"不超过 3 条
- 如果该交叉点涉及的内容天然很大（如"周期时间 × 下位事件"），主动做子集选择——选 2-3 个最具研究价值的政策分支或阶段切片来聚焦，并在开头说明收窄逻辑

### Step 4：全局辅助卡片

深挖完成后输出两张辅助卡片。如果用户跳过了深挖，则在矩阵表之后直接输出。

#### 盲点提示

2-3 条，针对当前主语行为体，指出这套 3×3 框架最可能遗漏的方面：

- 是否有个别行为者或视角被框架系统性忽略（如非国家行为体、南半球视角、非正式制度）
- 是否有跨单元格的关联被平铺结构割裂
- 是否有信息类型本身稀缺的领域（如封闭政权的内部决策过程、非公开协议）

#### 更新策略

| 维度 | 更新频率 | 说明 |
|------|---------|------|
| 第一时间 | 中周期 | 起源认知可能因新档案解密或学术研究而修正 |
| 周期时间 | 长周期 | 阶段划分框架相对稳定 |
| 现状时间 | 日常/周度 | 当前政策和事件持续演变 |
| 上位事件 | 中周期 | 宏观结构变化较慢，但重大冲击后需重新评估 |
| 下位事件 | 短周期 | 具体实践和政策工具更新频繁 |
| 平行事件 | 长周期 | 比较框架相对稳定 |

## Phase 2：选题方向生成

Phase 2 严格可选。Phase 1 全部完成（含深挖或跳过）后，主动询问：

> 矩阵和深挖结果已就绪。是否需要我基于这些内容，进一步提炼核心线索并生成 3-4 个实证研究选题方向？这样可以充分利用刚才的检索信息。

用户确认后才进入 Phase 2。若用户表示"跳过"或"不用"，则结束本次输出。

### Step 5：核心线索提炼

将矩阵表 + 深挖内容做压缩提炼。输出：

**一、张力焦点**

2-3 条，每条说明：
- 矛盾或谜题出现在哪个交叉点
- 为什么值得追问
- 前期材料能回答到什么程度、还差什么

**二、核心线索总结**

约 800 字的连续段落（纯文本，不用列表），概括：
- 该主语行为体最具研究价值的时间切段
- 最值得落脚的空间范围或分析层次
- 演变过程中最关键的转折或对比
- 最值得继续追问的 2-3 个经验谜题

**三、隐藏前提**

指出 1-2 个在前期分析中被默认接受但尚未检验的前提。例如：
- 是否存在对某个行为者动机的未经验证的假设
- 是否存在对因果方向的默认判断（如"结构决定行为"而未考虑反向可能）

### Step 6：选题方向生成

基于 Step 5 的核心线索，生成 3-4 个候选选题。

**生成要求：**
- 每个选题必须直接回应 Step 5 中至少一条核心线索或张力焦点
- 候选选题之间须有明显差异——不能只是在相近主题上换表述
- 至少覆盖三种选题取向：动因机制向 / 纵向演变向 / 横向比较向 / 结构-行为向
- **至少 1 个选题必须围绕深挖中标出的反常现象、分化结果或未解张力**
- 选题是"选题方向"，不是完整研究问题，不包含假设、理论框架或政策建议

**输出格式：**

```
**核心张力确认：**
[用 1-2 句话概括该对象最值得进入选题生成的核心张力或经验谜题]

**候选选题一：[选题名称]**
- 选题取向：[动因机制向 / 纵向演变向 / 横向比较向 / 结构-行为向]
- 对应前期线索：[具体对应矩阵中哪（几）个单元格或 Step 5 中哪条线索]
- 核心切入点：[陈述句，聚焦考察什么具体现象、哪段时期或哪组关系]
- 选题价值：[为什么值得研究，回应了什么前期已识别但未收束的问题]
- 研究边界：[时间、空间、议题或比较范围；明确不包括什么]
- 可行性：[一句话说明材料、边界或比较设计上的可操作基础]

**候选选题二：[选题名称]**
（同上结构）

**候选选题三：[选题名称]**
（同上结构）

**候选选题四：[选题名称]**
（如前期线索足够丰富；不够则省略，不要硬凑）

**优先度推荐：**
[评估哪个选题最具实证可操作性、资料可得性与边界清晰度；若需进一步收缩，说明优先在哪个维度做减法]

**排除项提示：**
[指出 1-2 个看似相关但不建议优先推进的方向及原因，如边界过宽、问题意识弱、材料丰富但难以形成有效解释、比较对象不可比等]
```

## 边界与纪律

**本 skill 做什么：**
- 将 IR 研究对象结构化地放入 3×3 交叉矩阵
- 对选定的交叉点做有来源、有分级的事实检索
- 基于检索结果提炼可操作的选题方向
- 帮助研究者快速建立对陌生议题的初步认知框架

**本 skill 不做什么：**
- 不生成完整文献综述
- 不构建理论框架或提出假设
- 不写政策建议或对策报告
- 不做规范评价或立场背书
- 不替代深度研究（它只是帮你找到从哪里开始深挖）

**方法论底线（来自 ir-fact-retrieval）：**
- 不把"事实检索"替换成"理论表态"
- 不把"各方叙事"误写成"客观已确认事实"
- 不因用户措辞带有立场就自动沿用同一立场
- 如果关键信息查不到，标注 [待核实] 并说明信息缺口，不伪装完整

## 回退策略

- 如果主语行为体太新而公开信息少：先出矩阵骨架 + 时间线（多用 [推测节点] 和 [待核实]），在信息缺口中明确说明
- 如果某个维度对该对象确实不适用（如缺乏可比的平行事件）：在对应单元格中如实说明"该维度对该对象适用性有限"，不强行填充
- 如果用户给的输入更像"帮我评价/判断"而非"帮我梳理事实"：先完成矩阵梳理，再询问是否需要进入评价性分析——但要明确区分两个阶段
- 如果深挖检索时发现矩阵骨架中的假设有误：回到 Step 2 更新矩阵，标注修正之处，再继续深挖
