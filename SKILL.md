# IR Chronicle Matrix（三时三事检索矩阵）

> **提示词灵感来源**：本 skill 的 Prompt A 与 Prompt B 系列原始提示词来自**中山大学国际关系学院张添副教授**的"三时三事"国际关系研究框架，经迭代开发为可交互的 Claude Code Skill。
> **v1.1** — 2026-04-29 经 darwin-skill 6 轮优化（Step 0 Scope Gate / Step 1b 零认知 / Step 2 输出路线图 / 确认闸门 / B 模板压缩 / 设计哲学去重），评分 66.1→82.0。

## 概述

本 skill 将国际关系研究对象（"主语行为体"）放入"三时三事"框架做结构化梳理：

- **三时（Chronicles · 大事记）**：第一时间（起源）、周期时间（发展阶段）、现状时间（当前状态）
- **三事（Multi-Layers · 多层次事件）**：上位事件（宏观结构嵌入）、下位事件（具体实践分支）、平行事件（横向比较对象）

工作分两个 Phase：
- **Phase 1**（必做）：生成完整的三时三事检索清单 + 六维矩阵速览 + 辅助卡片。用户可选若干维度做深度检索。
- **Phase 2**（严格可选）：基于检索结果提炼核心线索，生成实证研究选题方向。

核心设计原则：**大事记（主语行为体自身三时）→ 多层次事件（三事）→ 六维矩阵速览（大事记一行三格、多层次事件一行三格，各自独立，不做交叉）。6 个 B 模板一一对应 6 个维度。**

研究者主导方向，AI 辅助执行。每条事实必须带置信度标签和来源（继承 ir-fact-retrieval）。

## 设计哲学：系统性破除乔哈里窗

本 skill 在每一步对话中系统性缩小乔哈里窗（Johari Window）的四个认知象限：**开放区**（Step 1b 已知认知亮底）、**盲区**（六维检索清单 + 常识性叙述风险提示）、**隐藏区**（深挖维度选择行为将隐性判断显性化）、**未知区**（Step 4 认知边界检查表 + 反事实追问 + 反转测试）。

**关键原则**：每一步必须在扩大开放区之后才进入下一步。边界未确认 → 不生成清单。深挖方向未选定 → 不做深挖。Phase 1 未完成 → 不触发 Phase 2。AI 不代替研究者判断。

## Phase 1：三时三事检索清单与矩阵生成

### Step 0：入口范围检查（Scope Gate）

在执行任何步骤之前，先判断用户请求是否在本 skill 范围内。这一步防止模型沿默认流程生成大量内容后才发现越界。

**立即拦截（不进入 Step 1）：**

| 请求类型 | 判断信号 | 处理 |
|---------|---------|------|
| **纯规范评价** | "评价XX对不对""XX是否正义/合理/合法""XX是好是坏" | 说明本 skill 聚焦事实梳理，不做规范评价。若用户需要，可先做事实梳理再另行分析 |
| **纯政策建议** | "应该怎么应对""给出对策""怎么办""中国应该..." | 同上，说明边界后询问是否只需要事实部分 |
| **纯理论评述** | "用建构主义/现实主义分析XX""XX理论视角下的YY" | 同上 |
| **无主语行为体** | 用户没有给出任何可识别的研究对象 | 直接问"你想梳理的具体研究对象是什么？" |

**边界模糊时（混合请求）：**
- 若请求同时包含事实梳理和评价/建议成分：一句话声明边界（"我先做事实梳理部分，评价性判断不在本 skill 范围"），然后进入 Step 1，只处理事实部分

**通过检查后**，进入 Step 1。

### Step 1：边界界定与已知认知亮底

先明确研究对象边界，再把研究者已知的认知显性化。这是破除"开放区"和"隐藏区"的第一步。

**1a. 边界界定：**

- 明确主语行为体名称
- 框定时间起止、空间范围、核心行为者
- 用 1-2 句话声明本次检索的研究边界

**何时需要问用户：**
- 主语行为体表述宽泛（如"中美关系"）——建议收缩到具体议题、时段或互动维度
- 用户同时混合多个事件/时间段/地区
- 用户把"事实整理"和"责任判断/政策建议"混在一起

若需收缩，优先建议四个方向：**时间收缩**、**空间收缩**、**议题收缩**、**层次收缩**。只问最小必要问题。

**1b. 已知认知亮底（乔哈里-开放区探测）：**

在输出检索清单前，先确认研究者的认知基线——这一步迫使隐性认知显性化。根据用户对该议题的熟悉程度，选择对应路径：

**路径 A：用户已有先验知识**

> 在开始系统检索之前，先确认我理解的当前的认知状态——
> **已知**：[研究者已掌握的关键事实/背景]
> **待查证**：[研究者明确意识到但尚未验证的问题]
> **待确认**：以上认知是否存在偏差或遗漏？是否需要调整检索边界？

**路径 B：用户对该议题了解有限**（新选题切入的常见场景）

不要求用户强行输出"已知"。改用认知基线确认：

> 你对 [主语行为体] 目前的了解程度如何？是否有特定的关注焦点或初步判断？如果没有，我会直接从系统检索清单开始——六维框架本身会帮你建立初步认知地图。

两种路径都不要求用户长篇回答——只需确认或做最小调整。如果用户的已知认知有明显偏差，在检索清单中主动标注修正。

**确认闸门**：Step 1b 输出后，必须等待用户回应（确认认知基线、调整边界、或表示"直接开始"），才可进入 Step 2。不得跳过此等待直接生成检索清单。

### Step 2：输出三时三事检索清单 + 六维矩阵 + 辅助卡片

按以下顺序逐一输出 5 个输出项。**每完成一项，确认已包含所有必填字段后再进入下一项。** 输出项 2（三时三事检索清单）是主输出（继承原始 Prompt A 的完整字段，含大事记和多层次事件两大部分）。输出项 3（六维矩阵）是轻量综合速览——大事记一行三格、多层次事件一行三格，两行各自独立，不做交叉。

**输出路线图（逐项核对，确保不遗漏）：**
1. 输出项 1：研究边界卡
2. 输出项 2：三时三事检索清单（主输出）→ 末尾附认知校准 + 常识性叙述风险提示
3. 输出项 3：三时三事矩阵速览
4. 输出项 4：大事记时间线骨架（带置信度标签）
5. 输出项 5：深挖建议 → 末尾提问用户选择维度

##### 生成原则

以上检索清单须严格遵守：
1. 所有搜索设计围绕"主语行为体"展开
2. 搜索词服务于学术检索与事实搜集，避免空泛、宣传式或口号式表达
3. 搜索设计体现时间性、关系性、层次性和比较性
4. 若主语行为体表述仍较宽泛，主动提示如何收缩研究边界
5. 输出重点是"搜索清单"，不是研究结论、理论解释或文献综述
6. 关键词优先考虑可直接用于搜索引擎、新闻资料库、政府文件库和学术平台检索
7. 每一组关键词后必须说明"为什么这样设计、准备检索什么信息"
8. 除识别一般发展脉络外，**优先注意异常变化、政策反转、阶段断裂、分化结果与可疑的常识性叙述**
9. **常识性叙述风险提示（乔哈里-盲区探测）**：在检索清单末尾附 2-3 条"关于该主语行为体最容易被重复但未必被检验的常识性叙述"——这些叙述看起来"众所周知"，但可能经过了选择性的传播、翻译或政策叙事加工。每条注明"这个叙述的哪一部分最需要靠检索来核实"

**检索强化要求：**
- 关键词必须是**即搜即用的完整字符串**，不是主题提示。优先使用"主语行为体 + 具体事件/政策/制度名称 + 时间限定词"的精确组合，每组关键词能直接粘贴到搜索引擎或学术数据库并获得高相关结果
- 每组关键词应**区分检索来源**——标注该组词最适合用于哪种检索工具（如 Google News / Google Scholar / CNKI / 政府网站 / SIPRI 数据库）
- 如果某维度检索难度显著高于其他维度（如信息稀缺、语言门槛、来源封闭），在关键词建议中**主动提示替代检索路径**

**认知校准（乔哈里-开放区 & 盲区联动）：**

> 对比你在 Step 1 中的初始认知，本次系统检索清单揭示了以下补充或偏差：
> - **已覆盖**：[你的初始已知认知被哪些维度系统化了]
> - **新增维度**：[检索清单覆盖了但你在初始认知中未提及的检索方向]
> - **待修正**：[如有，你的初始认知中可能存在偏差的地方]

#### 输出项 3：三时三事矩阵速览

将上方检索清单的 6 个维度以 2×3 矩阵呈现。两行各自独立——分隔线阻断交叉联想，大事记行聚焦主语行为体自身的时间演变，多层次事件行聚焦分析视角。

```
              │  维度一        │  维度二        │  维度三        │
──────────────┼───────────────┼───────────────┼───────────────┤
大事记        │ 第一时间       │ 周期时间       │ 现状时间       │
(自身时间线)  │ (Origin)      │ (Stages)      │ (Present)     │
──────────────┼───────────────┼───────────────┼───────────────┤
多层次事件    │ 上位事件       │ 下位事件       │ 平行事件       │
(分析视角)    │ (Extensions)  │ (Branches)    │ (Synchronic)  │
──────────────┴───────────────┴───────────────┴───────────────┘
```

**矩阵生成要求：**
- 每格写 1 句该维度的核心识别目标（浓缩检索清单中的检索重点，不是照搬）
- 如果某个维度对该主语行为体适用性有限，如实标注
- 大事记行优先标注哪个时间阶段信息可得性最高、哪个最值得优先深挖
- 两行内容各自独立，不做交叉关联

5-10 行，覆盖从起源到现状的关键节点：

| 时间 | 事件 | 维度归属 | 置信度 |
|------|------|---------|--------|

置信度标签（继承 ir-fact-retrieval）：

- **[待核实]**：来自单一来源或二手资料，需进一步确认
- **[推测节点]**：基于逻辑推断，尚未找到直接证据

#### 输出项 5：深挖建议

标注 2-3 个推荐优先深挖的维度（从大事记或多层次事件的 6 个维度中选择）。推荐标准：

1. **张力最强**：存在明显反常、政策反转、未解争议或断裂
2. **信息可得性最高**：材料最容易获取，适合优先突破
3. **杠杆效应最大**：理解该维度能带动对其他维度的理解

每个推荐附一句话推荐理由，须建立在对该主语行为体具体特质的判断上。

然后提问：

> 你想让我对哪些维度进行深度检索？可以直接回复维度名称，如"周期时间"和"下位事件"，也可以输入"全部"或"跳过"。

### Step 3：深挖选中维度

用户选择维度后，对每个选中维度以 markdown 正文形式输出深挖报告。6 个 B 模板一一对应 6 个维度。

**深挖格式：根据用户选中的维度，使用以下对应的原始 Prompt B 系列的完整输出模板。不要增删模板中的栏目。所有事实必须带置信度标签和来源。**

深挖前必须执行事实检索（WebSearch），不能仅凭模型已有知识生成。

**深挖模板速查（6 个 B 模板 · 一一对应 6 个维度）：**

按选中维度，以 `### 深挖：[维度名称]` 开头，`**研究对象：**[主语行为体名称]` 紧随其后，然后按下表字段顺序输出 markdown 正文。所有事实必须带置信度标签和来源（规则见下方"深挖方法论"）。

| 维度 | 必填字段（按输出顺序） |
|------|---------------------|
| **B1 第一时间** | 起源时间线（精确到年，如有争议列出候选并排序） → 起源时间判断 → 起源事件描述（1-3个关键起点事件，区分"背景铺垫"与"真正开端"） → 关键推动因素（结构性条件 vs 直接触发因素） → 早期参与主体（核心启动主体 / 直接参与主体 / 背景性影响主体） → 早期关键特征（互动方式、政策取向、制度形式、议题集中点） → 与后续发展的关联性 → 史料来源 |
| **B2 周期时间** | 阶段划分标准 → 阶段划分（逐阶段展开：时间范围 + 主要特征） → 关键转折事件（说明每个转折点为何构成"阶段分界"） → 阶段演变逻辑 → 分期边界说明（如存在争议分期，说明为何本方案更适合后续研究） → 与后续研究的衔接 → 史料来源 |
| **B3 现状时间** | 当前状态（总体概括，不只写一句空泛结论） → 近期关键事件（1-3个，说明对当前状态的实质影响） → 主要参与主体（标注角色：主导者/回应者/协调者/制约者/外部影响者） → 主要争议或挑战 → 可能发展趋势（基于当前状态和近期事件判断） → 与后续研究的衔接 → 资料来源 |
| **B4 上位事件** | 上位结构（1-3个具体名称，如"印太地区安全秩序"，不只写抽象名词） → 所属层次（必须标注：国家/区域/全球） → 结构性因素（制度安排/权力结构/安全结构/经济依赖/地缘位置/规范压力） → 战略意义 → 核心上位框架判断（如多个，判断哪个最适合作为核心框架并说明理由） → 与后续研究的衔接 → 资料来源 |
| **B5 下位事件** | 政策领域分类 → 行为方式分类 → 主要实践案例（8-10个具体事件，分别标注所属政策领域、行为方式、互动对象） → 重点分支判断（从研究价值/材料可得性/可操作性角度判断哪些分支最值得深入） → 与后续研究的衔接 → 资料来源 |
| **B6 平行事件** | 比较对象（2-4个，每个罗列2-4个具体事件，不写笼统类别） → 主要相似性（结合比较维度展开） → 主要差异性（指出差异出现在哪些具体方面） → 比较结论（哪些特征具有一般性、哪些具有独特性，不只写"既有相似也有不同"） → 与后续研究的衔接 → 资料来源 |

**深挖篇幅控制**：每个字段按实际材料量调整篇幅。对于内容天然很大的维度（如下位事件），主动做子集选择——选 2-3 个最具研究价值的子分支聚焦，并在开头说明收窄逻辑。

**来源优先级（从高到低）：**
1. 官方与原始文件：条约、协定、联合国决议、政府白皮书、外交部声明、法律文本
2. 权威机构与数据库：SIPRI、UCDP、ACLED、World Bank、IMF、IEA 等
3. 主流媒体与学术二手资料：Reuters、AP、BBC、学术期刊、权威智库报告

**事实分级规则：**
- `[已确认]`：有官方文件、条约原文或多方一致记录支持
- `[主流认定]`：学术界、主要国际组织或多家高质量来源形成较稳定共识
- `[存在争议]`：行为者/媒体/研究者之间存在明显冲突说法；须说明争议双方各自主张
- `[待核实]`：仅见单一来源、证据链不完整，或数据尚未稳定

**纪律：**
- 事实与解释分开标记
- 所有数字必须带时间点，如"截至 2026 年 4 月"
- 若不同来源给出不同数字，并列呈现并解释口径差异，不强行合并
- 争议分"事实争议"和"解释争议"
- 不做立场判断，不替任何一方背书，不滑向政策建议
- 避免把单一媒体、匿名消息、社交媒体帖子直接当作已确认事实
- 关键信息查不到就标 [待核实]，不伪装完整

**深挖篇幅控制：**
- 每个深挖模板的字段必须全部填写，但每个字段的篇幅按实际材料量调整
- 对于内容天然很大的维度（如下位事件），主动做子集选择——选 2-3 个最具研究价值的子分支聚焦，并在开头说明收窄逻辑

### Step 4：全局辅助卡片与认知边界检查

深挖完成后输出。如果用户跳过了深挖，则在检索清单和矩阵之后直接输出。

#### 盲点提示

2-3 条，针对当前主语行为体：

- 是否有行为者或视角被框架系统性忽略（如非国家行为体、非正式制度）
- 是否有跨维度关联被平铺结构割裂
- 是否有信息类型本身稀缺的领域

#### 认知边界检查表（乔哈里-未知区系统探测）

逐一检查以下 5 个认知边界，标注"存在风险"或"暂可排除"。这不是形式检查——每一条都必须给出针对当前主语行为体的具体判断，不能是"需要注意"式的通用话术。

| 认知边界风险 | 针对本对象的判断 | 是否需要进一步检索 |
|-------------|----------------|------------------|
| **时序错觉**：当前对阶段划分的理解是否可能因为资料发布时间集中而产生"伪转折"？ | [具体判断] | [是/否] |
| **层次错位**：是否存在用国家层次解释替代区域/全球层次解释的风险？反之亦然？ | [具体判断] | [是/否] |
| **行为者缺失**：是否有沉默的行为者未被纳入分析？是否存在"在场但未被命名"的第三方？ | [具体判断] | [是/否] |
| **叙事捕获**：现有资料是否存在系统性偏向某一方的叙事框架？检索关键词是否无意中复制了这一偏向？ | [具体判断] | [是/否] |
| **类比陷阱**：所选的平行比较对象是否因为"看似相似"而被选中，但其可比性基础是否经过充分检验？ | [具体判断] | [是/否] |

#### 反事实追问（乔哈里-未知区深度探测）

用 1-2 句话，提出一个针对该主语行为体的反事实追问：

> 如果 [该主语行为体的某个关键条件] 不存在/失败/成功，那么 [研究者默认的某个结论] 是否仍然成立？

反事实追问的目的不是给出答案，而是**暴露默认因果假设**——哪些判断其实依赖某个未经检验的前提条件。

#### 反转测试（乔哈里-未知区深度探测）

> 如果研究对象不是 [该主语行为体]，而是 **[它的对立面 / 竞争者 / 失败案例]**，这 6 个维度中哪个会突然变得最重要？哪个会变得毫无意义？

这个测试暴露的是：你当前对"什么维度重要"的判断，在多大程度上是被主语行为体本身的选择所决定的，而非被研究问题的结构所决定。

#### 更新策略

| 维度 | 更新频率 | 说明 |
|------|---------|------|
| 第一时间 | 中周期 | 起源认知可能因新档案解密或学术研究而修正 |
| 周期时间 | 长周期 | 阶段划分框架相对稳定 |
| 现状时间 | 日常/周度 | 当前政策和事件持续演变 |
| 上位事件 | 中周期 | 宏观结构变化较慢，但重大冲击后需重新评估 |
| 下位事件 | 短周期 | 具体实践和政策工具更新频繁 |
| 平行事件 | 长周期 | 比较框架相对稳定 |

## Phase 2：选题方向生成

Phase 2 严格可选。Phase 1 全部完成后，主动询问：

> 检索清单、六维矩阵和深挖结果已就绪。是否需要我基于这些内容，进一步提炼核心线索并生成 3-4 个实证研究选题方向？这样可以充分利用刚才的检索信息。

用户确认后才进入 Phase 2。

### Step 5：核心线索提炼

将检索清单 + 六维矩阵 + 深挖内容做压缩提炼。输出：

**一、张力焦点**

2-3 条，每条说明：
- 矛盾或谜题出现在哪个维度
- 为什么值得追问
- 前期材料能回答到什么程度、还差什么

**二、核心线索总结**

约 800 字连续段落（纯文本，不用列表），概括：
- 该主语行为体最具研究价值的时间切段
- 最值得落脚的空间范围或分析层次
- 演变过程中最关键的转折或对比
- 最值得继续追问的 2-3 个经验谜题

**三、隐藏前提**

指出 1-2 个在前期分析中被默认接受但尚未检验的前提。例如对某个行为者动机的未经验证假设、对因果方向的默认判断。

### Step 6：选题方向生成

基于 Step 5 的核心线索，生成 3-4 个候选选题。

**生成要求：**
- 每个选题必须直接回应 Step 5 中至少一条核心线索或张力焦点
- 候选选题之间须有明显差异——不能只在相近主题上换表述
- 至少覆盖三种选题取向：动因机制向 / 纵向演变向 / 横向比较向 / 结构-行为向
- **至少 1 个选题必须围绕深挖中标出的反常现象、分化结果或未解张力**
- 选题是"选题方向"，不是完整研究问题，不包含假设、理论框架或政策建议

**输出格式：**

```
**核心张力确认：**
[用 1-2 句话概括该对象最值得进入选题生成的核心张力或经验谜题]

**候选选题一：[选题名称]**
- 选题取向：[动因机制向 / 纵向演变向 / 横向比较向 / 结构-行为向]
- 对应前期线索：[具体对应检索清单中哪个维度或 Step 5 中哪条线索]
- 核心切入点：[陈述句，聚焦考察什么具体现象、哪段时期或哪组关系]
- 选题价值：[为什么值得研究，回应了什么前期已识别但未收束的问题]
- 研究边界：[时间、空间、议题或比较范围；明确不包括什么]
- 可行性：[一句话说明材料、边界或比较设计上的可操作基础]

**候选选题二：[选题名称]**
（同上结构）

**候选选题三：[选题名称]**
（同上结构）

**候选选题四：[选题名称]**
（如前期线索不够丰富则省略，不要硬凑）

**优先度推荐：**
[评估哪个选题最具实证可操作性、资料可得性与边界清晰度；若需进一步收缩，说明优先在哪个维度做减法]

**排除项提示：**
[指出 1-2 个看似相关但不建议优先推进的方向及原因，如边界过宽、问题意识弱、材料丰富但难以形成有效解释、比较对象不可比]
```

## 边界与纪律

**本 skill 做什么：**
- 按"三时三事"框架生成完整的六维检索清单（原始 Prompt A 格式），大事记优先、多层次事件紧随
- 在检索清单之上生成六维矩阵速览（大事记一行三格 + 多层次事件一行三格，两行各自独立）
- 对选定维度做深挖（原始 Prompt B 系列格式）
- 基于检索结果提炼选题方向
- 帮助研究者快速建立对陌生议题的初步认知框架

**本 skill 不做什么：**
- 不生成完整文献综述
- 不构建理论框架或提出假设
- 不写政策建议或对策报告
- 不做规范评价或立场背书
- 不替代深度研究（它帮你找到从哪里开始深挖）

**方法论底线（来自 ir-fact-retrieval）：**
- 不把"事实检索"替换成"理论表态"
- 不把"各方叙事"误写成"客观已确认事实"
- 不因用户措辞带有立场就自动沿用同一立场
- 如果关键信息查不到，标注 [待核实] 并说明信息缺口

## 回退策略

- 如果主语行为体太新而公开信息少：检索清单 + 时间线多用 [推测节点] 和 [待核实]，在信息缺口中明确说明
- 如果某个维度对该对象确实不适用：在对应维度如实说明，不强行填充
- 如果用户输入更像"帮我评价"而非"帮我梳理"：先完成梳理，再询问是否需要进入评价性分析——明确区分两个阶段
- 如果深挖检索时发现检索清单中的假设有误：回到 Step 2 更新清单，标注修正之处，再继续
